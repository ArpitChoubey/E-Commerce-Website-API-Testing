# 🛒 E-Commerce Website API Testing

## 🔗 Website Under Test

This project focuses on API testing of the e-commerce platform:
👉 https://sparklecartonline.com/

---

## 📌 Project Overview

This repository contains complete API testing artifacts for an e-commerce website, covering cart functionalities such as adding items, updating cart, removing items, and refreshing cart data.

The goal of this project is to ensure:

* Functional correctness of APIs
* Data integrity and validation
* Performance and reliability
* Security and error handling

---

## 🔌 API Endpoints Covered

### 🟢 GET – Refresh Cart

`https://sparklecartonline.com/?wc-ajax=get_refreshed_fragments`

### 🟠 POST – Add to Cart

`https://sparklecartonline.com/?wc-ajax=add_to_cart`

### 🔵 PUT – Update Cart

`https://sparklecartonline.com/?wc-ajax=update_cart`

### 🟡 PATCH – Update Cart Item

`https://sparklecartonline.com/?wc-ajax=update_cart_item`

### 🔴 DELETE – Remove from Cart

`https://sparklecartonline.com/?wc-ajax=remove_from_cart`

## 📂 Repository Contents

This repository includes the following documents:

* 📊 **SparkleCart Website API Test Cases.xlsx**
  → Contains  detailed API test cases covering functional, negative, and edge scenarios

* 📄 **Requirement Traceability Matrix (RTM)**
  → Maps 30 requirements (R1–R30) with all 152 test cases

* 📘 **SparkleCart API Documentation.docx**
  → Detailed API specifications including endpoints, request/response details

* 📝 **SparkleCart API Test Plan Document.docx**
  → Covers scope, strategy, tools, risks, and execution plan
